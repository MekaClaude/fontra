## Known Technical Debt (Not Yet Addressed)

### 1. Source Map Errors (Cosmetic)
**Files**: Various webpack-bundled JS files
**Error**: `Erreur dans les liens source : Error: URL constructor: is not a valid URL.`
**Impact**: DevTools sourcemap loading fails — doesn't affect runtime
**Priority**: LOW
**Fix**: Review webpack sourcemap config, ensure valid URLs

### 2. CSS Parse Warnings (Cosmetic)
**Files**: `editor.html` inline styles
**Error**: `Déclaration attendue, mais « ; » trouvé.` / `Propriété inconnue`
**Impact**: None — browser skips invalid declarations
**Priority**: LOW
**Fix**: Review inline CSS in `editor.html` for non-standard syntax

### 3. fontra-pak Submodule Modified Content
**Files**: `fontra-pak/` directory
**Issue**: Git shows "modified content" in submodule
**Impact**: Can cause confusion in version control
**Priority**: MEDIUM
**Fix**: Commit submodule changes separately or sync with upstream

### 4. Ignored `_version.py` File
**File**: `src/fontra/_version.py`
**Issue**: Auto-generated by setuptools-scm, gitignored
**Impact**: Version doesn't persist in repo — relies on build-time generation
**Priority**: LOW (by design)
**Note**: Manual edits to this file are overwritten on reinstall

## Improvement Proposals

### Proposal A: Add Linting Configuration
**Goal**: Catch undeclared variables, duplicate code, and common errors before runtime
**Action**:
- Add ESLint with `no-undef`, `no-redeclare`, `no-duplicate-imports` rules to `src-js/`
- Add a pre-commit hook or CI check
**Effort**: Medium
**Impact**: HIGH — Would have caught bugs #1, #2, #3 automatically

### Proposal B: Add Circular Dependency Detection
**Goal**: Detect circular imports at build time
**Action**:
- Add `madge --circular` to webpack build process
- Or add `circular-dependency-plugin` to webpack config
**Effort**: Low
**Impact**: HIGH — Would have caught bug #1 at bundle time

### Proposal C: Consolidate Duplicate Code
**Goal**: Reduce code duplication across the codebase
**Areas to review**:
- Geometry functions (`computeApex`, `pointInTriangle`, `isDegenerate`) are duplicated across 3+ files
- Panel styling patterns repeated across 10+ panel files
- Violation scanning logic duplicated between panel and tool
**Effort**: Medium-High
**Impact**: Medium — Easier maintenance, fewer inconsistency bugs

### Proposal D: Add Automated Testing
**Goal**: Ensure critical paths don't regress
**Action**:
- Add Playwright E2E tests for editor loading
- Test that all panels can be constructed
- Test that tools can be initialized
**Effort**: High
**Impact**: HIGH — Catches regressions before they reach users

### Proposal E: Improve Local Plugin API
**Goal**: Make it easier for local plugins to access Fontra APIs without duplication
**Action**:
- Expose `@fontra/` imports via import map in `editor.html`
- OR provide a global `FontraAPI` object on `window` for local plugins
- Document the local plugin API contract
**Effort**: Medium
**Impact**: Medium — Better plugin ecosystem

## Build Process Improvements

### Current Build Flow
1. Manual version bump (4 files)
2. `npm install && npm run bundle`
3. `pip install -r requirements.txt`
4. `python -m PyInstaller FontraPak.spec --clean`
5. Manual copy to versioned folder

### Proposed: Automated Build Script
Create `build-fontra.ps1` that:
1. Detects current GMT+1 timestamp
2. Updates all version files
3. Runs npm bundle
4. Runs PyInstaller
5. Creates versioned output folder
6. Runs test-export.py
7. Reports build success/failure
