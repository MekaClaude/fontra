## 2. Code Audit Findings

### Software Architecture & Extensibility
- **Plugin System**: The current plugin initialization in `editor.js` (`initPlugins`) is functional but minimal. It relies on a `plugin.json` and a specific entry point.
  - *Improvement*: Formalize the Plugin API with a dedicated manager class that handles lifecycles (init, activate, deactivate, destroy) and provides scoped access to Fontra APIs.
- **Circular Dependencies**: Some modules (like `edit-tools-triangle-guardian.js`) use lazy imports to avoid circular dependencies.
  - *Improvement*: Refactor shared geometry or utility functions into a separate `core` module to break these cycles and allow for cleaner static imports.

### UI/UX & Consistency
- **Icon Strategy**: The codebase currently mixes Phosphor Icons with custom Fontra SVGs.
  - *Improvement*: Standardize all tool icons to use either the custom Fontra set (in `/images/`) or Phosphor Icons, but not both intermittently.
- **Micro-Animations**: The UI is functional but feels "static".
  - *Improvement*: Add subtle transitions to panel expansions and tool switching to make the interface feel more premium and responsive.
- **Sidebar Management**: Sidebar width is managed through a fixed constant `MIN_SIDEBAR_WIDTH`.
  - *Improvement*: Implement a more flexible layout engine (likely using CSS Grid/Flexbox with resizable handles) that persists specific widths per panel to localStorage.

### Performance
- **Asset Loading**: Icons and images are fetched as individual SVG files.
  - *Improvement*: Bundle core icons into an SVG sprite or use an icon font to reduce HTTP requests during initial load.

## 3. Future Ideas & Improvements

### A. Intelligent Design Assistance (Pro Advice)
- **Visual Overlays**: Instead of just text advice in the `ProAdvicePanel`, implement visual guides that draw over the glyph (e.g., showing where overshoots should be or highlighting stem consistency).
- **Auto-Correction Suggestions**: Add a "Fix" button in the advice panel that applies common optical corrections (like adding 1-2% overshoot to round glyphs) automatically.

### B. Enhanced Workflow Tools
- **Contextual Inspector**: A floating or dedicated "Selection" panel that changes its options based on what is selected (contour, point, anchor, component), reducing the need to look through multiple panels.
- **Glyph DNA Tracking**: A system that remembers the relationships between glyphs (e.g., 'o' informs 'c', 'e', 'd', 'p', 'q') and prompts the user to update related glyphs when the foundation glyph changes.

### C. Advanced Visualization
- **Curvature Comb & Speedpunk Integration**: (Already planned/in progress) Deeply integrate high-end curvature analysis into the default visualization layers.
- **Comparison Mode**: A split-screen or overlay mode to compare the current glyph across different axes or masters in real-time.
